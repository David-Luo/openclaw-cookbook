# SKILLS.md — 技能注册与调用说明

## 说明

本文件记录当前工作区可用的自定义 Skill。
每个 Skill 是一个独立的能力单元，Agent 可按需调用。

## 已注册技能

### /architecture-review
- 用途：对架构方案进行结构化评审
- 触发：用户提到"架构评审"、"方案评审"
- 输出：评审报告，包含优点、风险、改进建议

### /adr
- 用途：生成架构决策记录（Architecture Decision Record）
- 触发：用户提到"记录决策"、"ADR"
- 输出：标准 ADR 格式文档（背景 → 决策 → 理由 → 后果）

### /compare
- 用途：技术方案对比分析
- 触发：用户提到"对比"、"选型"、"A还是B"
- 输出：多维度对比表格 + 推荐结论

### /deep-research
- 用途：深度技术调研
- 触发：用户提到"调研"、"研究一下"
- 输出：调研报告，包含现状分析、方案对比、推荐路径

## 添加新技能

在 `skills/` 目录下创建对应的 SKILL.md 文件，遵循 OpenClaw Skill 标准格式，
然后在本文件中注册即可。
