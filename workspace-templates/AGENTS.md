# AGENTS.md — 多 Agent 协作与流程编排

## 会话启动

- 读取 SOUL.md 确认身份和行为准则
- 读取 USER.md 了解用户偏好和上下文
- 检查当前工作目录和项目状态

## 通用规则

- 修改文件前先阅读完整内容，理解上下文再动手
- 不主动提交 Git，除非用户明确要求
- 遇到不确定的问题，先询问再行动
- 每次任务完成后，简要总结做了什么和为什么
- 保持代码风格与项目现有风格一致

## 架构设计任务

当用户请求架构设计时：

1. 先澄清需求和约束条件
2. 列出备选方案（至少两个）
3. 用表格对比各方案的优劣
4. 给出推荐方案及理由
5. 输出 Mermaid 或 ASCII 架构图
6. 标注已知风险和待验证假设

## 代码审查任务

当用户请求 Code Review 时：

1. 先理解变更的业务目的
2. 关注点优先级：正确性 > 安全性 > 性能 > 可维护性 > 风格
3. 给出具体修改建议，而非笼统评论
4. 区分"必须改"和"建议改"

## 文档编写任务

当用户请求编写技术文档时：

1. 确认文档的目标读者
2. 采用适合读者的语言和深度
3. 关键决策记录使用 ADR 格式：背景 → 决策 → 理由 → 后果
4. 架构文档必须包含图表

## 故障排查任务

当用户请求排查问题时：

1. 先收集现象：错误信息、日志、复现步骤
2. 形成假设并排优先级
3. 逐一验证，排除法缩小范围
4. 找到根因后给出修复方案和预防建议

## 工具使用偏好

- 架构图：优先使用 Mermaid，复杂场景用 draw.io
- 文档格式：Markdown
- 版本管理：Git，遵循 Conventional Commits
- 代码搜索：优先用语义搜索，再用正则搜索
